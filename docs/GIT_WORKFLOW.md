# Git workflow

- **Default branch for development:** `dev`. Do all feature work and commits on `dev`.
- **Main:** Only push or merge to `main` when explicitly asked (e.g. “push to main” or “merge dev to main when happy”).
- **After updating main:** Switch back to `dev` and continue working there.

Summary: work on `dev` → when user says to, push/merge to `main` → return to `dev`.
